#### Remote View SaaS 구조 개선

**배경**

* 원격제어 서비스 특성상 상태 조회와 상태 전이가 빈번
* 운영 과정에서 성능 저하 및 상태 불일치 이슈 반복 발생

**역할**

* 문제 재현 조건 정의 및 원인 분석 주도
* 모듈 단위 설계 방향을 제안하고 구조 개선을 직접 이끌어 반복 성능 이슈를 해결
* 동료 개발자의 설계·구현 리뷰를 통해 작업 방향을 정렬하고 동일 패턴 결함 재발을 차단

**접근 및 해결**

* 계층형 조회 구조의 성능 병목을 분석 — 그룹 depth에 비례해 재귀 DB 쿼리가 반복되는 N+1과 캐시 광역 무효화가 원인
* 회사 단위 계층을 DB 1회로 조회한 뒤 메모리에서 재귀 탐색하도록 전환 — 재귀 DB 쿼리 N회를 1회로 축소
* 그룹별 캐시를 회사 단위 통합 캐시로 재설계해 적중률·무효화 범위를 단순화
* 명령 처리와 read model을 분리하고, 캐시 갱신(Rebuild)과 단일 복구 지점을 분리하여 장애 발생 시 시스템 전파를 차단

**결과**

* 그룹 1만 개 조회 기준 약 20초 → 수백ms로 단축
* 상태 일관성 대응 구조 정립 — 이후 구조 개선도 무중단으로 수행

#### 인증 아키텍처 리팩토링

**배경**

* 레거시 인증 구조의 복잡도로 인한 유지보수 어려움
* 플랫폼별 인증 흐름과 응답 처리 방식 차이 및 확장 요구 증가

**역할**

* Spring Security 표준 아키텍처 적용 주도
* 인증/인가 흐름 재설계 및 커스텀 필터 구조 정비

**접근 및 해결**

* 요청 유형별로 나뉜 2개 필터 체인을 단일 인증 파이프라인으로 통합 — 요청이 복합적이라 분리가 강제되던 경로는 커스텀 필터로 한 체인 안에서 처리
* 3종 인증 토큰(기본 JWT·웹뷰어 JWT·별도 AccessToken)과 경로별 3갈래 인증 정책을 Spring Security 표준 필터 흐름으로 재구성
* 산재된 예외 처리를 단일 핸들러로 통합하고, 인증과 인가의 관심사를 분리

**결과**

* 인증 로직 변경 시 영향 범위 감소
* 신규 인증 방식(SSO 등) 추가 시 구조적 부담 최소화
