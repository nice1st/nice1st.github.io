#### Remote View SaaS 구조 개선

**배경**

* 원격제어 서비스 특성상 상태 조회와 상태 전이가 빈번
* 운영 과정에서 성능 저하 및 상태 불일치 이슈 반복 발생

**역할**

* 문제 재현 조건 정의 및 원인 분석 주도
* 모듈 단위 설계 방향을 제안하고 구조 개선을 직접 이끌어 반복 성능 이슈를 해결
* 동료 개발자의 설계·구현 리뷰를 통해 작업 방향을 정렬하고 동일 패턴 결함 재발을 차단

**접근 및 해결**

* 계층형 조회 구조의 성능 병목을 분석 — 비즈니스 로직 내 재귀 DB 호출과 캐시 광역 무효화가 원인
* 명령 처리와 read model을 분리하고, Redis 기반 캐시로 조회 경로 단순화
* 식별자 기준 캐시 구조로 재설계해 변경 영향 범위를 해당 식별자로 한정
* 상태 변경 이벤트를 기준으로 read model을 점진적으로 동기화
* 캐시 갱신(Rebuild)과 단일 복구 지점을 분리하여 장애 발생 시 시스템 전파를 차단

**결과**

* 식별자 기반 캐시 도입으로 조회 응답 속도 수십배 향상
* 상태 일관성 대응 구조 정립 — 이후 구조 개선도 무중단으로 수행

* Spring Security 표준 아키텍처 적용 주도

* SSO 기반 인증 흐름을 기존 구조에 자연스럽게 통합할 수 있도록 설계

**결과**

* 인증 로직 변경 시 영향 범위 감소
* 신규 인증 방식(SSO 등) 추가 시 구조적 부담 최소화
